Summary

HTML5 semantic elements are elements with a meaning. These elements are used to clearly describe and define the different parts of a web page to the browser being used and the developers using them. Semantics are primarily used to help technologies read and interpret your webpage, help computers make sense of the content you have created, help other programmers understand the structure of your webpages, and for easier use internationally since English is not widely spoken around the world. Some of the main semantic tags that one should know for document outlines include, but are not limited to: header, nav, aside, section, article, footer, etc.. Each semantic element has its purpose and place in a web page, but at the end of the day, they are all used to organize the content that one is trying to convey, ultimately allowing for a more efficient way to share and reuse data.
